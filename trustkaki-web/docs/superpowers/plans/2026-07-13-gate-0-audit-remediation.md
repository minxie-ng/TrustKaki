# Gate 0 Audit Remediation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the independent authorization, actor identity, RLS, transaction, timeout, maintainability, and verification findings before TrustKaki resumes feature development or production deployment promotion.

**Architecture:** Every browser agent request carries only an authorized `seniorId` and bounded operation input; a server-only context repository loads authoritative senior state before orchestration. Relationship-scoped RLS uses locked private helper functions, while authenticated public RPC commands perform caregiver case updates and demo reset atomically. Existing large UI and persistence files are split along their current responsibilities after behavior is protected by regression tests.

**Tech Stack:** Next.js 16 App Router, TypeScript, React 19, Zod 4, Supabase Auth/Postgres/RLS/RPC, Vitest, ESLint.

---
